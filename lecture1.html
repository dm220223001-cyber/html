1. Что такое веб-разработка?

Веб-разработка — это процесс создания веб-сайтов и веб-приложений для сети Интернет. Она включает в себя несколько направлений: frontend-разработка (клиентская часть, то есть то, что видит пользователь в браузере) и backend-разработка (серверная часть, которая обрабатывает данные и взаимодействует с базами данных). В данном курсе мы сфокусируемся на frontend-разработке — создании интерфейсов с помощью HTML, CSS и JavaScript.

🏗️ HTML

Строительные леса сайта. Определяет структуру и содержание страницы: заголовки, абзацы, изображения, кнопки.

🎨 CSS

Дизайн и стилизация. Отвечает за цвета, шрифты, расположение элементов, анимацию и адаптивность.

⚡ JavaScript

Интерактивность и логика. Обрабатывает клики, отправку форм, загружает данные без перезагрузки страницы.

2. Что такое HTML?

HTML (HyperText Markup Language) — это стандартизированный язык разметки документов для просмотра веб-страниц в браузере. Основная задача HTML — описать структуру веб-страницы: какие элементы на ней присутствуют, как они расположены и какую смысловую роль выполняют. HTML задаёт только каркас документа, а не его поведение или внешний вид.

HTML был разработан британским учёным Тимом Бернерсом-Ли приблизительно в 1986–1991 годах во время его работы в ЦЕРНе (Европейская организация по ядерным исследованиям). Первая версия HTML была создана для обмена научными документами между исследователями. Сегодня HTML является основой всего Всемирной паутины.

📌 Интересный факт: Первый веб-сайт в истории был запущен 6 августа 1991 года. Он объяснял, что такое Всемирная паутина, как создать свой веб-сервер и как пользоваться браузером. Этот сайт до сих пор доступен по адресу info.cern.ch.

3. Что такое CSS?

CSS (Cascading Style Sheets) — это язык стилей, с помощью которого оформляют веб-страницы. Он определяет, как должны отображаться элементы HTML, включая макет, цвета, шрифты, отступы и другие визуальные аспекты. CSS используют, чтобы отделить содержание веб-страницы от её оформления. Это позволяет легко изменять дизайн всего сайта, не трогая HTML-код.

CSS был предложен Хоконом Виум-Ли в 1994 году, а первая версия спецификации была опубликована в 1996 году. До появления CSS дизайн веб-страниц создавался с помощью таблиц и атрибутов HTML, что было крайне неудобно.

4. Что такое JavaScript?

JavaScript — это язык программирования, с помощью которого сайты делают интерактивными. Он позволяет кнопкам реагировать на клики, формам проверять введённые данные, меню плавно открываться при наведении курсора, а также асинхронно загружать данные с сервера без перезагрузки страницы (технология AJAX).

JavaScript был создан в 1995 году Брэнданом Эйхом, работающим в компании Netscape. Изначально он назывался Mocha, затем LiveScript, и только потом получил имя JavaScript. Несмотря на схожесть названия, JavaScript не имеет прямого отношения к языку Java.

⚠️ Важно понимать: HTML, CSS и JavaScript работают вместе, но выполняют разные задачи. Нельзя «написать сайт на JavaScript» без HTML, и нельзя «стилизовать» сайт без CSS. Это три взаимодополняющих технологии.

5. Структура HTML-документа

Любой HTML-документ имеет строгую иерархическую структуру. Рассмотрим её подробно:

<!DOCTYPE html> <html lang="ru"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Название страницы (отображается во вкладке браузера)</title> <meta name="description" content="Краткое описание страницы для поисковых систем"> <link rel="stylesheet" href="styles.css"> <script src="script.js" defer></script> </head> <body> <!-- Здесь находится всё видимое содержимое страницы --> <header>Шапка сайта</header> <main>Основное содержание</main> <footer>Подвал сайта</footer> </body> </html>

Пояснение основных элементов:

  • <!DOCTYPE html> — объявление типа документа. Оно указывает браузеру, что это HTML5-документ. Должно быть самым первым в файле.
  • <html lang="ru"> — корневой элемент страницы. Атрибут lang указывает основной язык содержимого.
  • <head> — контейнер для метаданных. Всё, что здесь находится, не отображается на самой странице, но важно для браузера и поисковых систем.
  • <meta charset="UTF-8"> — задаёт кодировку символов. UTF-8 поддерживает все символы Unicode, включая русские буквы и эмодзи.
  • <meta name="viewport"> — управляет отображением на мобильных устройствах. width=device-width устанавливает ширину окна просмотра равной ширине экрана устройства.
  • <title> — заголовок страницы, отображаемый во вкладке браузера и в результатах поиска.
  • <body> — контейнер для всего видимого содержимого страницы.
  • <!-- комментарий --> — комментарии не отображаются на странице и служат для пояснения кода разработчикам.

6. Основные теги HTML

6.1. Заголовки (<h1> — <h6>)

Теги заголовков определяют иерархию информации на странице. <h1> — самый важный заголовок (обычно один на странице), <h2> — подзаголовки разделов, <h3> — подразделы и так далее. Использование правильной иерархии заголовков важно для SEO-оптимизации.

<h1>Главный заголовок страницы</h1> <h2>Раздел 1</h2> <h3>Подраздел 1.1</h3> <h2>Раздел 2</h2> <h3>Подраздел 2.1</h3> <h4>Подподраздел 2.1.1</h4>

6.2. Параграфы (<p>)

Тег <p> (от англ. paragraph) используется для создания абзацев текста. Браузер автоматически добавляет отступы до и после каждого абзаца.

<p>Это первый абзац текста. Он будет отделён от следующего абзаца отступом.</p>

6.3. Списки

Существует три основных типа списков:

  • Маркированный список (<ul>) — элементы отмечаются маркерами (точками, кружками).
  • Нумерованный список (<ol>) — элементы нумеруются автоматически.
  • Список определений (<dl>) — состоит из терминов и их описаний.
<!-- Маркированный список --> <ul> <li>Первый пункт</li> <li>Второй пункт</li> <li>Третий пункт</li> </ul> <!-- Нумерованный список --> <ol> <li>Шаг 1</li> <li>Шаг 2</li> <li>Шаг 3</li> </ol>

6.4. Ссылки (<a>)

Тег <a> (от англ. anchor — якорь) создаёт гиперссылки. Атрибут href (hypertext reference) указывает адрес перехода.

<!-- Внешняя ссылка --> <a href="https://www.google.com">Перейти на Google</a> <!-- Внутренняя ссылка --> <a href="about.html">О нас</a> <!-- Ссылка на email --> <a href="mailto:info@example.com">Написать нам</a> <!-- Ссылка на телефон --> <a href="tel:+71234567890">Позвонить</a> <!-- Открыть в новой вкладке --> <a href="https://example.com" target="_blank">Открыть в новой вкладке</a>
💡 Полезный атрибут target="_blank": открывает ссылку в новой вкладке. Это удобно для внешних сайтов, чтобы пользователь не покидал вашу страницу. Однако для внутренних ссылок лучше этого не делать.

6.5. Изображения (<img>)

Тег <img> — самостоятельный (не требует закрывающего тега). Обязательные атрибуты: src (источник изображения) и alt (альтернативный текст).

<!-- Базовое изображение --> <img src="photo.jpg" alt="Описание фотографии"> <!-- С указанием размеров --> <img src="photo.jpg" alt="Описание" width="300" height="200"> <!-- С подсказкой при наведении --> <img src="photo.jpg" alt="Описание" title="Всплывающая подсказка">
⚠️ Атрибут alt — обязательный! Он отображается, если изображение не загрузилось, и используется скринридерами для незрячих пользователей. Это важно для доступности (a11y) и SEO.

6.6. Форматирование текста

Для выделения текста используются специальные теги:

<strong>Жирный текст (смысловое выделение)</strong> <b>Жирный текст (визуально)</b> <em>Курсив (смысловое ударение)</em> <i>Курсив (визуально)</i> <mark>Выделенный цветом текст</mark> <small>Мелкий текст</small> <del>~~Зачёркнутый текст~~</del> <ins>Вставленный текст</ins> <sub>Подстрочный текст</sub> <sup>Надстрочный текст</sup>

7. Атрибуты тегов

Атрибуты предоставляют дополнительную информацию об элементах. Они записываются внутри открывающего тега в формате имя="значение".

id

Уникальный идентификатор элемента. Должен быть уникальным на всей странице.

<div id="header">

class

Класс элемента. Может быть одинаковым у нескольких элементов. Позволяет группировать элементы для стилизации.

<p class="text-center">

style

Прямое CSS-стилизование элемента (инлайновые стили). Рекомендуется использовать редко.

<div style="color: #20b2aa;">

data-*

Пользовательские атрибуты для хранения дополнительных данных, доступных через JavaScript.

<div data-id="123">

8. Создание первой веб-страницы

Давайте создадим нашу первую веб-страницу шаг за шагом:

  1. Откройте любой текстовый редактор (Блокнот, VS Code, Sublime Text, Notepad++).
  2. Создайте новый файл и сохраните его с именем index.html.
  3. Скопируйте следующий код в файл:
<!DOCTYPE html> <html lang="ru"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Моя первая страница</title> </head> <body> <h1>Добро пожаловать на мой первый сайт!</h1> <p>Это моя первая веб-страница, созданная с помощью HTML.</p> <h2>Мои любимые технологии:</h2> <ul> <li>HTML — для структуры</li> <li>CSS — для красоты</li> <li>JavaScript — для интерактивности</li> </ul> <p> <a href="https://www.w3schools.com" target="_blank"> Узнать больше о веб-разработке </a> </p> <img src="https://picsum.photos/300/200" alt="Случайное изображение"> </body> </html>
  1. Сохраните файл и откройте его в любом браузере (дважды кликните по файлу).
  2. Поздравляю! Вы создали свою первую веб-страницу.

9. Инструменты разработчика

Современные браузеры имеют встроенные инструменты для разработчиков, которые помогут вам отлаживать HTML, CSS и JavaScript.

  • Как открыть: нажмите клавишу F12 или Ctrl+Shift+I (Windows) / Cmd+Option+I (Mac).
  • Вкладка Elements (Элементы): позволяет просматривать и редактировать HTML-структуру страницы в реальном времени.
  • Вкладка Styles (Стили): показывает все CSS-правила, применённые к выбранному элементу.
  • Вкладка Console (Консоль): для выполнения JavaScript-кода и просмотра ошибок.
  • Вкладка Network (Сеть): показывает все сетевые запросы, загрузку файлов, изображений.
💡 Совет: В инструментах разработчика можно редактировать HTML и CSS прямо в браузере и сразу видеть изменения. Это отличный способ экспериментировать и учиться, не боясь сломать исходный код.

10. Полезные ресурсы

  • MDN Web Docs (developer.mozilla.org) — самая полная и авторитетная документация по веб-технологиям на русском языке.
  • W3Schools (w3schools.com) — отличный ресурс с примерами и интерактивными упражнениями.
  • HTML Validator (validator.w3.org) — официальный валидатор HTML для проверки правильности кода.
  • Can I Use (caniuse.com) — проверка поддержки различных веб-технологий в разных браузерах.
  • CodePen (codepen.io) — песочница для экспериментов с HTML, CSS и JS прямо в браузере.
