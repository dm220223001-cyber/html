1. Что такое семантическая верстка?

Семантическая верстка — это подход к созданию HTML-кода, при котором теги выбираются не по внешнему виду, а по их смысловому значению. Вместо универсального контейнера <div> используются специализированные теги, которые точно описывают тип содержимого.

✅ Преимущества семантической верстки

• Улучшает SEO — поисковые системы лучше понимают структуру страницы
• Повышает доступность (a11y) — скринридеры правильно интерпретируют контент
• Облегчает поддержку кода — разработчики быстрее ориентируются в структуре
• Улучшает кросс-браузерную совместимость

❌ Несемантическая верстка

• Использование <div> и <span> для всего подряд
• Поисковые системы хуже индексируют страницу
• Скринридеры не могут правильно озвучить структуру
• Код становится «кашей» из вложенных контейнеров

2. Основные семантические теги

2.1. Структурные семантические теги

ТегОписаниеПример использованияОсобенности
<header>Шапка сайта или разделаЛоготип, название сайта, основные контакты, навигацияМожет повторяться на странице (в разных разделах)
<main>Основное уникальное содержимое страницыГлавный контент статьи, каталог товаров, форма обратной связиДолжен быть только один на странице
<footer>Подвал сайта или разделаКопирайт, контакты, ссылки на политику, карта сайтаОбычно содержит контактную информацию
<nav>Блок навигационных ссылокГлавное меню, хлебные крошки, оглавление страницыПредназначен для основных навигационных блоков
<article>Независимая, самодостаточная часть контентаНовость, пост в блоге, комментарий, карточка товараМожет быть скопирован и распространён отдельно
<section>Группирует связанный по смыслу контентРаздел преимуществ, блок с услугами, группа новостейДолжен иметь заголовок внутри
<aside>Контент, связанный с основным, но не являющийся его частьюБоковая панель, блок с рекламой, ссылки на похожие статьиЧасто используется для дополнительной информации
<!-- Пример семантической структуры страницы --> <body> <header> <h1>Мой блог</h1> <nav> <a href="/">Главная</a> <a href="/about">О нас</a> <a href="/contacts">Контакты</a> </nav> </header> <main> <article> <h2>Название статьи</h2> <p>Содержание статьи...</p> </article> <section> <h2>Похожие статьи</h2> <p>Список похожих материалов...</p> </section> </main> <aside> <h3>Реклама</h3> <p>Рекламный блок...</p> </aside> <footer> <p>© 2024 Мой блог. Все права защищены.</p> </footer> </body>
💡 Важно: Теги <section> и <article> всегда должны иметь заголовок (обычно <h1>—<h6>) внутри себя, чтобы обозначить тему раздела.

3. Форматирование текста

Для придания тексту смыслового и визуального выделения используются специальные теги:

ТегЗначениеВизуальный эффектПример использования
<strong>Смысловая важность, серьезное предупреждениеЖирный<strong>Важно! Не забудьте сохранить</strong>
<b>Визуальное выделение без смысловой нагрузкиЖирный<b>Ключевое слово</b> в тексте
<em>Смысловое ударение, эмфазисКурсивЭто <em>очень</em> важно
<i>Визуальный курсив без смысловой нагрузкиКурсивИностранное слово <i>c'est la vie</i>
<mark>Выделение цветом (как маркером)Желтый фон<mark>Важный фрагмент</mark>
<small>Мелкий текст (юридические оговорки, примечания)Меньший размер шрифта<small>*Акция действует до 31.12</small>
<del>Удаленный текстЗачёркнутый<del>старая цена 1000₽</del>

<sup>Надстрочный текстВыше строки, меньший размерE=mc<sup>2</sup> (формула Эйнштейна)

4. Работа со ссылками

Гиперссылки создаются с помощью тега <a> (anchor — якорь). Основной атрибут — href (hypertext reference), указывающий адрес перехода.

Типы путей для ссылок

Тип путиСинтаксисПримерКогда использовать
Абсолютный (полный)протокол://домен/путьhttps://example.com/page.htmlСсылки на внешние сайты
Относительный (текущая папка)имя_файла.html или ./page.html или ./page.htmlВнутренние ссылки в пределах одной папки
Относительный (уровень выше)../../page.htmlСсылки из подпапок в родительские папки
Относительный (корневой)//page.htmlУниверсальные ссылки от корня сайта

Атрибуты тега <a>

АтрибутОписаниеПример
hrefАдрес ссылки (обязательный атрибут)href="https://example.com"
target="_blank"Открывает ссылку в новой вкладкеtarget="_blank"
rel="noopener"Защита при открытии в новой вкладке (безопасность)rel="noopener noreferrer"
titleВсплывающая подсказка при наведенииtitle="Подробнее о компании"
downloadПредлагает скачать файл вместо переходаdownload="filename.pdf"
<!-- Внешняя ссылка с защитой --> <a href="https://www.google.com" target="_blank" rel="noopener noreferrer"> Перейти на Google </a> <!-- Внутренние ссылки --> <a href="about.html">О нас (текущая папка)</a> <a href="../index.html">На главную (на уровень выше)</a> <a href="/catalog/page.html">Каталог (от корня сайта)</a> <!-- Специальные ссылки --> <a href="mailto:info@example.com">Написать письмо</a> <a href="tel:+71234567890">Позвонить</a> <a href="#section-1">Перейти к разделу 1 (якорь на этой же странице)</a> <a href="document.pdf" download>Скачать PDF</a>
🔒 Рекомендация по безопасности: При использовании target="_blank" всегда добавляйте rel="noopener noreferrer". Это защищает вашу страницу от потенциальных атак через открытую ссылку.

5. Вставка изображений

Тег <img> является самозакрывающимся (не требует закрывающего тега) и используется для встраивания изображений на страницу.

Атрибуты тега <img>

АтрибутОбязательныйОписаниеПример
src✅ ДаПуть к файлу изображения (абсолютный или относительный)src="images/photo.jpg"
alt✅ ДаАльтернативный текст (для SEO и доступности)alt="Описание изображения"
title❌ НетВсплывающая подсказка при наведении мышиtitle="Подсказка"
width / height❌ НетРазмеры изображения в пикселяхwidth="300" height="200"
loading❌ НетЛенивая загрузка (lazy/eager)loading="lazy"
⚠️ Атрибут alt — обязательный! Он отображается, если изображение не загрузилось, используется поисковыми системами для индексации и важен для людей с нарушениями зрения (скринридеры зачитывают alt-текст). Для декоративных изображений используйте alt="".
<!-- Базовое изображение с обязательными атрибутами --> <img src="images/logo.png" alt="Логотип компании"> <!-- Изображение с указанием размеров --> <img src="images/photo.jpg" alt="Пейзаж" width="600" height="400"> <!-- Изображение с подсказкой и ленивой загрузкой --> <img src="images/banner.jpg" alt="Рекламный баннер" title="Нажмите для перехода" loading="lazy"> <!-- Изображение-ссылка --> <a href="https://example.com"> <img src="images/button.jpg" alt="Кнопка перехода"> </a>

<td>Обычная ячейка с данными (table data)Содержит основные данные таблицы

7. Сравнение семантической и несемантической верстки

АспектСемантическая версткаНесемантическая верстка
Используемые теги<header>, <nav>, <main>, <article>, <section>, <footer><div>, <span> (без смысловой нагрузки)
Читаемость кодаВысокая — сразу понятно, где какой блокНизкая — нужно анализировать классы и структуру
SEO-оптимизацияПоисковые системы лучше понимают структуру и важность контентаПоисковые системы хуже индексируют страницу
Доступность (a11y)Скринридеры правильно озвучивают структуруСкринридеры не понимают назначение блоков
Поддержка кодаРазработчики быстро ориентируются в структуреКод сложнее поддерживать и модифицировать
Будущее технологииСовременный стандарт, активно развиваетсяУстаревший подход, не рекомендуется
<!-- ❌ НЕСЕМАНТИЧЕСКАЯ ВЕРСТКА (плохо) --> <div class="header"> <div class="logo">Логотип</div> <div class="menu"> <div class="menu-item">Главная</div> <div class="menu-item">О нас</div> </div> </div> <div class="content"> <div class="post">Статья...</div> </div> <div class="footer">Подвал</div> <!-- ✅ СЕМАНТИЧЕСКАЯ ВЕРСТКА (хорошо) --> <header> <div class="logo">Логотип</div> <nav> <a href="/">Главная</a> <a href="/about">О нас</a> </nav> </header> <main> <article>Статья...</article> </main> <footer>Подвал</footer>
📝 Итог лекции:
• Семантическая верстка использует теги по их смысловому назначению, а не по внешнему виду.
• Основные семантические теги: <header>, <main>, <footer>, <nav>, <article>, <section>, <aside>.
• Для форматирования текста используются <strong>, <em>, <blockquote>, <time> и другие.
• Ссылки создаются тегом <a> с атрибутом href.
• Изображения вставляются через <img> с обязательными атрибутами src и alt.
• Таблицы строятся с помощью <table>, <tr>, <td>, <th>.
• Семантическая верстка улучшает SEO, доступность и облегчает поддержку кода.
