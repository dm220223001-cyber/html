1. Что такое DOM-дерево?

DOM (Document Object Model) — это программный интерфейс, который браузер создаёт при загрузке HTML-страницы. DOM представляет документ в виде древовидной структуры, где каждый HTML-тег становится узлом-объектом. DOM предоставляет JavaScript доступ к структуре страницы, позволяя читать, изменять и удалять элементы в реальном времени, без перезагрузки.

🌳 Структура DOM-дерева

Каждый HTML-элемент — это узел (node). Узлы образуют иерархию: родительский → дочерние. Корнем дерева является объект document, а корневым элементом — <html>.

🔧 Зачем нужен DOM?

• Динамическое изменение содержимого страницы
• Реагирование на действия пользователя (события)
• Анимация элементов
• Создание интерактивных компонентов

Визуальное представление DOM-дерева

document └── html ├── head │ ├── title │ └── meta └── body ├── header │ ├── h1 │ └── nav ├── main │ ├── article │ └── section └── footer

2. Поиск элементов в DOM

Чтобы что-то сделать с элементом, его сначала нужно найти. В современном JavaScript существует несколько способов.

Способы поиска элементов

МетодВозвращаетСкоростьПример
document.getElementById('id')Один элемент или null?, Очень быстрый?, let header = document.getElementById('main-header');?
document.querySelector('селектор')Первый подходящий элемент или null?, Быстрый?, let btn = document.querySelector('.btn-primary');?
document.querySelectorAll('селектор')Статическая коллекция (NodeList)?, Средний?, let items = document.querySelectorAll('.menu-item');?
document.getElementsByClassName('класс')Живая коллекция HTMLCollection?, Быстрый?, let elements = document.getElementsByClassName('active');?
document.getElementsByTagName('тег')Живая коллекция HTMLCollection?, Быстрый?, let paragraphs = document.getElementsByTagName('p');?
// Поиск по id (уникальный идентификатор) let header = document.getElementById('main-header'); // Поиск по CSS-селектору (первый элемент) let firstButton = document.querySelector('.btn'); // Поиск всех элементов по селектору let allButtons = document.querySelectorAll('.btn'); // Навигация по DOM let parent = element.parentElement; // родительский элемент let children = element.children; // дочерние элементы let nextSibling = element.nextElementSibling; // следующий соседний элемент let previousSibling = element.previousElementSibling; // предыдущий соседний элемент
💡 Рекомендация: Используйте querySelector и querySelectorAll — они универсальны и работают с любыми CSS-селекторами. Это современный и удобный стандарт.

3. Изменение содержимого и стилей

Найдя элемент, можно менять его содержимое и внешний вид.

Работа с содержимым

Свойство/МетодОписаниеБезопасность
innerHTMLЧитает/записывает HTML-разметку внутри элемента⚠️ Риск XSS
textContentЧитает/записывает только текст (без HTML-тегов)✅ Безопасно
innerTextПохож на textContent, но учитывает стили (медленнее)✅ Безопасно

Управление стилями

// Изменение инлайновых стилей (через camelCase) element.style.backgroundColor = '#20b2aa'; element.style.fontSize = '18px'; element.style.padding = '10px 20px'; // Получение вычисленных стилей let bgColor = getComputedStyle(element).backgroundColor; // Управление классами (рекомендуемый способ) element.classList.add('active'); // добавить класс element.classList.remove('hidden'); // удалить класс element.classList.toggle('open'); // переключить класс (есть — удалить, нет — добавить) element.classList.contains('active'); // проверить наличие класса // Работа с атрибутами element.getAttribute('data-id'); element.setAttribute('data-id', '123'); element.hasAttribute('data-id'); element.removeAttribute('data-id');
💡 Совет: Для изменения внешнего вида всегда используйте classList. Определяйте CSS-классы в отдельном файле, а JavaScript пусть только переключает их. Это делает код чище и производительнее.

5. Работа с событиями

События — это сигналы, которые браузер генерирует в ответ на действия пользователя или системы.

Способы назначения обработчиков

СпособПримерНедостатки
Атрибут onclick<button onclick="alert('Click')">Смешивает HTML и JS, нельзя назначить несколько обработчиков
Свойство onclickbutton.onclick = function() {...}Нельзя назначить несколько обработчиков
addEventListenerbutton.addEventListener('click', () => {...})✅ Рекомендуемый способ
// addEventListener — универсальный способ button.addEventListener('click', () => { console.log('Кнопка нажата!'); }); // Можно добавить несколько обработчиков на одно событие button.addEventListener('click', () => { console.log('Первый обработчик'); }); button.addEventListener('click', () => { console.log('Второй обработчик'); }); // Удаление обработчика function handler() { console.log('Клик'); } button.addEventListener('click', handler); button.removeEventListener('click', handler);

Объект события (event)

element.addEventListener('click', (event) => { console.log(event.type); // тип события ("click") console.log(event.target); // элемент, на котором произошло событие console.log(event.currentTarget); // элемент, на котором висит обработчик console.log(event.clientX); // координата X относительно окна console.log(event.clientY); // координата Y относительно окна event.preventDefault(); // отмена действия по умолчанию event.stopPropagation(); // остановка всплытия события });

Популярные типы событий

КатегорияСобытияОписание
Мышьclick, dblclick, mousedown, mouseup, mouseenter, mouseleave, mousemoveДействия с мышью
Клавиатураkeydown, keyup, keypressНажатия клавиш
Формыsubmit, change, input, focus, blur, resetДействия с формами и полями ввода
Документ/окноDOMContentLoaded, load, resize, scroll, beforeunloadСобытия загрузки, прокрутки, изменения размера
Drag & Dropdragstart, drag, dragend, dragenter, dragleave, dragover, dropПеретаскивание элементов

Всплытие событий (Event Bubbling) и делегирование

// Всплытие: событие сначала срабатывает на самом глубоком элементе, // затем поднимается вверх по иерархии DOM // Делегирование событий — обработчик на родителе для всех дочерних элементов document.querySelector('.menu').addEventListener('click', (event) => { if (event.target.classList.contains('menu-item')) { console.log('Нажат пункт меню:', event.target.textContent); } });
🔍 Делегирование событий: Это мощный паттерн, позволяющий назначить один обработчик на родительский элемент для множества дочерних. Особенно полезен для динамически добавляемых элементов.

6.1. Модальное окно

6.3. Слайдер изображений

let images = ['img1.jpg', 'img2.jpg', 'img3.jpg']; let currentIndex = 0; let sliderImg = document.getElementById('slider'); let prevBtn = document.getElementById('prev'); let nextBtn = document.getElementById('next'); function updateImage() { sliderImg.src = images[currentIndex]; } nextBtn.addEventListener('click', () => { currentIndex = (currentIndex + 1) % images.length; updateImage(); }); prevBtn.addEventListener('click', () => { currentIndex = (currentIndex - 1 + images.length) % images.length; updateImage(); }); // Автоматическое перелистывание setInterval(() => { currentIndex = (currentIndex + 1) % images.length; updateImage(); }, 3000);

7. Публикация сайта: хостинг и домены

Когда сайт готов, его нужно сделать доступным в интернете. Для этого требуется хостинг (место на сервере) и домен (адрес сайта).

Бесплатные платформы для статических сайтов

ПлатформаОсобенностиАвтоматическая сборка
GitHub PagesБесплатно, интеграция с GitHub, поддержка собственного доменаИз репозитория
NetlifyDrag-and-drop загрузка, формы, функции, SSL-сертификатИз репозитория
VercelОтлично для Next.js, превью для каждого pull requestИз репозитория
Cloudflare PagesБыстрая CDN, аналитика, неограниченные запросыИз репозитория
📡 Как опубликовать на GitHub Pages (пошагово):
1. Зарегистрируйтесь на github.com
2. Создайте новый репозиторий (кнопка «New»)
3. Назовите репозиторий ваш_логин.github.io
4. Загрузите файлы проекта (перетащите папку с index.html, CSS, JS)
5. Нажмите «Commit changes»
6. Перейдите в Settings → Pages → Branch: main → Save
7. Через 1-2 минуты сайт будет доступен по адресу https://ваш_логин.github.io
🚀 Альтернативный способ — Netlify (drag-and-drop):
1. Зарегистрируйтесь на netlify.com
2. Перетащите папку с проектом мышкой в специальную область на сайте
3. Через секунду сайт опубликован с уникальным адресом (например, random-name.netlify.app)
4. Можно подключить свой домен и настроить HTTPS автоматически
⚠️ Важно: Для статических сайтов (HTML, CSS, JS) бесплатных хостингов достаточно даже для небольших коммерческих проектов. Платный хостинг нужен, если требуется серверная логика (PHP, Node.js, базы данных).
📝 Итог лекции:
• DOM — древовидная структура, отражающая HTML-документ.
• Элементы ищутся через getElementById, querySelector, querySelectorAll.
• Содержимое меняется через innerHTML (опасно) или textContent (безопасно).
• Стили управляются через classList (рекомендуется) или style.
• События обрабатываются через addEventListener. Делегирование событий упрощает код.
• Можно создавать интерактивные компоненты: модальные окна, слайдеры, бургер-меню, аккордеоны.
• Готовый сайт публикуется на GitHub Pages, Netlify, Vercel — это быстро и бесплатно.
