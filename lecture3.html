1. Что такое CSS?

CSS (Cascading Style Sheets) — это язык стилей, который отвечает за внешний вид веб-страницы. Если HTML создаёт структуру (каркас) сайта, то CSS определяет, как эта структура будет выглядеть: цвета, шрифты, расположение элементов, отступы, анимацию и адаптивность под разные устройства.

CSS был предложен Хоконом Виум-Ли в 1994 году, а первая версия спецификации была опубликована в 1996 году. До появления CSS дизайн веб-страниц создавался с помощью таблиц и атрибутов HTML, что было крайне неудобно и трудно поддерживаемо.

📌 Ключевая идея CSS: разделение структуры (HTML) и оформления (CSS). Это позволяет легко менять дизайн всего сайта, не трогая HTML-код, и использовать одни и те же стили для множества страниц.

📐 Блочная модель

Каждый элемент на странице — это прямоугольный блок с content, padding, border и margin. Понимание блочной модели — основа вёрстки.

2. Синтаксис CSS

CSS-правило состоит из селектора (указывает, к каким элементам применить стили) и блока объявлений (сами стили в фигурных скобках). Каждое объявление — это пара «свойство: значение».

селектор { свойство: значение; свойство: значение; }
/* Пример CSS-правила */ h1 { color: #20b2aa; /* сине-морской цвет текста */ font-size: 32px; /* размер шрифта 32 пикселя */ text-align: center; /* выравнивание по центру */ }
⚠️ Важно: Каждое объявление заканчивается точкой с запятой (;). Если забыть точку с запятой, следующее свойство может не примениться или вызвать ошибки в интерпретации стилей.

5. Каскадность, наследование и специфичность

Название CSS расшифровывается как Cascading Style Sheets — каскадные таблицы стилей. Понимание каскада критически важно для правильной работы стилей.

5.1. Каскад

К одному элементу могут применяться правила из разных источников (браузерные стили, стили пользователя, стили автора), и они накладываются друг на друга. Правила, объявленные позже (ниже по коду), переопределяют более ранние.

5.2. Наследование

Многие свойства (особенно связанные с текстом: цвет, шрифт, размер) автоматически передаются от родительских элементов к дочерним.

/* Свойство color наследуется потомками */ body { color: #ffffff; /* Весь текст на странице станет белым */ font-family: Arial, sans-serif; } /* Свойство margin НЕ наследуется */ .parent { margin: 20px; /* Дочерние элементы не получат этот отступ */ }

• px хорош для границ (border) и теней

9. Блочная модель CSS (Box Model)

Каждый элемент на веб-странице — это прямоугольный блок, который состоит из четырёх частей:

十六章Линия вокруг padding и content, видимая/невидимая, цветная/th>
Часть блочной моделиОписаниеCSS-свойство
Content (Контент)Содержимое элемента (текст, изображения)width, height (по умолчанию применяются только к контенту)
Padding (Внутренний отступ)Пространство между контентом и границей. Прозрачный фон.padding-top, padding-right, padding-bottom, padding-left или padding: 10px;
Border (Граница)border-width, border-style, border-color или border: 1px solid #fff;
Margin (Внешний отступ)Пространство между границей элемента и соседними элементами. Прозрачный.Ю章margin-top, margin-right, margin-bottom, margin-left или margin: 10px;
/* Пример блочной модели */ .card { width: 300px; /* Ширина контента */ height: 200px; /* Высота контента */ padding: 20px; /* Внутренние отступы со всех сторон */ border: 2px solid #20b2aa; /* Граница */ border-radius: 12px; /* Скругление углов */ margin: 30px auto; /* Внешние отступы (авто — центрирование по горизонтали) */ } /* Альтернативная блочная модель — width включает padding и border */ .card-alternative { box-sizing: border-box; /* Теперь width = padding + border + content width */ }
💡 Совет по блочной модели:
Используйте box-sizing: border-box; для всех элементов. Это упрощает расчёты ширины, потому что padding и border включаются в общую ширину элемента, а не добавляются к ней.

📝 Итог лекции

Основные выводы:
• CSS отвечает за внешний вид веб-страницы, отделяя оформление от структуры HTML.
• Существует три способа подключения CSS: инлайновые стили, встроенный CSS и внешний файл (рекомендуемый).
• Селекторы позволяют точно указывать, какие элементы стилизовать; важнейшие типы — по тегу, классу, id.
• Каскадность, наследование и специфичность определяют, какие стили будут применены в случае конфликта.
• Для задания размеров используются px (абсолютные), rem/em (относительные) и проценты.
• Цвет можно задавать через ключевые слова, HEX, RGB/RGBA, HSL/HSLA.
• Блочная модель — основа вёрстки: каждый элемент имеет content, padding, border и margin.
